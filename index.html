Projects

second-brain
Go React private

A structured personal-knowledge service that any AI agent can query via MCP (Model Context Protocol). Stores preferences, policies, vendor ratings, and order history across domains — food, coding, cabs, services. Agents call get_briefing() for fast, token-frugal context. Built with Go, Postgres + pgvector, MCP over HTTP/SSE, and a React dashboard for feedback and metrics. Includes sensitivity classification, provenance tracking, and idempotent markdown import.

sales-saathi
Python private

Live Hindi sales-call coaching tool. Two people role-play a loan sales call with push-to-talk; the system transcribes via local whisper-cpp (Metal-accelerated, no cloud), whispers real-time coaching nudges, then produces a scored scorecard across 7 dimensions with evidence-verified quotes. Supports live coaching and recorded-call upload. Runs entirely locally — no API keys, no cloud.

farm-pro
Python Kotlin private

Bilingual (English + Hindi) Android app and FastAPI backend that surfaces IMD weather data, severe-weather nowcasts, and agricultural advisory bulletins to farmers in Kanpur Dehat, UP. Scrapes IMD data and parses DAAS PDF bulletins. Every card shows its source link and freshness badge. Offline-first with WorkManager background sync. Currently in internal testing.

photo-gallery
Python ML Cloudflare private

Turns event photos into a deployed site where guests click a face to see every photo that person appears in. Uses RetinaFace + ArcFace for detection and recognition, agglomerative clustering to group faces. Guests can collaboratively name people. Deployed on Cloudflare Pages + R2 + KV with password protection. Used for real weddings and events.

lomasa-cli
Go shipped

Local AI agent manager — like nvm, but for open-source AI agents. Install, configure, and launch tools like Ollama and OpenCode. No API keys. No cloud. Everything runs locally. Named after the sage Lomasa from the Mahabharata.

About

I'm a builder who doesn't stick to one platform. Android in Kotlin, iOS in Swift, browser extensions in JS, CLIs in Go, backends in Python, ML pipelines when the problem calls for it. If it solves a problem, I'll learn whatever stack gets it done.

Right now I'm deep in the AI/LLM space — not just using these tools, but building infrastructure around them. second-brain is an MCP-native memory service for AI agents. sales-saathi runs local whisper + LLM for real-time Hindi coaching. lomasa-ai is where the open-source work lives.

I build for domains I care about — fintech (loan origination for underserved borrowers), agritech (weather advisories for farmers in UP), and tools that make daily life better. Two projects are on public app stores. The rest solve problems I actually had.
